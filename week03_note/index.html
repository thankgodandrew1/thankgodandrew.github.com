Mobile UX Design

UX which is explicitly know as user experience is a primary aspect of web development, it refers to the positive experiences web clients/users experience during the use of mobile devices. Mobile UX design focuses strongly on:

  • Learnability: How easily first-time users complete basic tasks.
  • Efficiency: How quickly users perform basic tasks once they've learned the design.
  • Memorability: The ability of users to remember how to use the system.
  • Errors: The amount and severity of errors users make and how easily they can correct them.
  • Satisfaction: How pleasant the experience of using the design was.

The reading on mobile user experience (UX) presents 10 tried and tested UX design principles, which are as follows:

  • Content Visualization: applying this principle requires the designer to identify and select only the content that is most valuable to display, and eliminate contents that is not of the highest importance.
  • Intuitive Navigation: the application of this principle result to easy navigation of your website, users find it easy to search for anything they are looking for, and also they also know were to go to if they can't find what they are looking for.
  • Touchscreen Target Sizes: another important principle to consider is target sizes, a good consideration of the size and space of an action button is important.
  • Provide User Control: user control is giving your users the control to do what they want or the option to undo something. For example, if a user deletes something by mistake and you give an 'undo' option, you're giving the user control and that in the long run gives he user a great satisfaction.
  • Legible Text Content: providing readability to users is a key design principles, some things to consider to improve legibility are: spacing, layout, and font size
  • Clear Interface Element: this principles requires designers to have sufficient contrast between content and background, one good tool designers csn use to check for accessibility issue is webaim.org
  • Hand Position Controls: Hand positions should be kept in mind when placing controls on a mobile design. Research findings on how people hold and interact with their mobile phones found that 49% of people rely on one thumb. This statistic is useful to keep in mind when designing to accommodate for the thumb's reach zone.
  • Minimize Data Input: minimizing the need to enter data in your designs is vital it also provides a good user experience.
  • Create a Seamless Experience: considering this aspect of design provides an experience that is devoid of any mistakes, delays, or setbacks as users journey through your product.
  • Test Your Design: On the reading it states "The key to any successful product is to continuously test and optimize." This would help designers detect any problem on the product.

The application of these principles would result to a good mobile user experience (UX)

Web APIs

API in it's full form is Apllication Programming Interface, APIs are built and made available in programming languages to allow developers to create complex functionality more easily. APIs removes complex codes away from developers, it provides some easier syntax to use in it's place.

There are two categories of APIs in client-side Javascript. Browser APIs and Third-party APIs. The Browser API are built into a web browser, they are able to expose data from the browser and surrounding computer environment, infact, talking of a great importance a Browser API can extend the functionality of a web browser. While a Third-party API are APIs provided by third parties (NOT the browser) example, Twitter, Google, Facebook, e.t.c to allow developers to access their functionality via JavaScript and use it on their site

How do APIs work? --- They are based on objects. They have recognizable entry points. They often use events to handle changes in state and. They have additional security mechanisms where appropriate

setTimeout and setInterval

As programmers we may decide to execute a function not immediately, but at a later time. That's called "scheduling a call."

There two methods for it:

  • setTimeout allows us to run a function once after the interval of time.
  • setInterval allows us to run a function repeatedly, starting after the interval of time, then repeating continuously at that interval.

The synatax for the setTimeout and setInterval appears to be the same.

  • Methods setTimeout(func, delay, ...args) and setInterval(func, delay, ...args) allow us to run the func once or regularly after delay milliseconds.
  • To cancel the execution, we should call clearTimeout/clearInterval with the value returned by setTimeout/setInterval.

Nested setTimeout are good and it's a more flexible alternative to setInterval, allowing uss to set the time between executions to be more precise. Zero delay scheduling with setTimeout(func, 0) is used to schedule the call "as soon as possible, but right after the current script is complete.
