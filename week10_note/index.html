Functions, Execution Contexts, and the Call Stack

Functions are blocks of code that can be defined and invoked multiple times. They can accept parameters as inputs and return a value as output. Functions help in breaking down complex code into smaller, reusable blocks, improving code organization, and reducing duplication.

Execution contexts refer to the environment in which JavaScript code is executed. It contains information about the code being executed, including variables, functions, and their values. Every time a function is called, a new execution context is created to manage its own set of variables and functions.

The call stack is a data structure that tracks the current execution context of the program. It keeps track of the order in which functions are called and the current execution context. The call stack operates in a last-in, first-out (LIFO) order, meaning the most recently called function is executed first, and the last function called is executed last.

Here's an example of a simple function:

function addNumbers(num1, num2) { return num1 + num2; } var result = addNumbers(5, 10); console.log(result); // Output: 15

In this example, the function addNumbers takes two parameters num1 and num2, and returns their sum. The function is then called with arguments 5 and 10, and the result is stored in a variable result which is logged to the console.

Callbacks and Higher-Order Functions in JavaScript

Callbacks and higher-order functions are essential concepts in JavaScript that are used to create more flexible and dynamic code.

A callback function is a function that is passed as an argument to another function and is executed when that function completes. Callbacks are often used for asynchronous operations, such as fetching data from an API, where the response may not be immediate.

Here's an example code that depicts the use of a callback function:

  • function fetchData(callback) {
        fetch('https://api.example.com/data')
            .then(response => response.json())
            .then(data => callback(data))
    }
  • fetchData(data => {
        console.log(data);
    });

Higher-order functions are functions that take other functions as arguments or return a function as their output. They provide a way to abstract and generalize code, making it more reusable and modular.
